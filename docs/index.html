# Wraps a data frame in a CIL-styled flextable ready to embed in the page.
# Usage: data |> cil_table()
cil_table <- function(data) {
  data |>
    flextable() |>
    set_table_properties(width = 1, layout = "autofit") |>
    border_remove() |>
    bold(bold = TRUE, part = "header") |>
    hline(
      border = fp_border(color = "#0b0b0b", width = 0.75),
      part   = "all"
    ) |>
    padding(
      padding.top = 0, padding.bottom = 4,
      padding.left = 0, padding.right = 16,
      part = "header"
    ) |>
    htmltools_value() |>
    div(class = "scroll-output")
}

This branch is for a brief RADAR report example that includes examples of typical elements:

  • Code blocks
  • Inline code
  • Hyperlinks
  • Graphs
  • Interactive maps
  • Block quotes

The code block above contains per-block rendering options, denoted by #|. Look, code formatting! Required packages are then written to a character vector reqd_pkgs which is used to compare against already-installed R packages using the setdiff() function.

Once packages are installed, they may be used by calling the library() function. The common packages used here are:

  • flextable: used for tables, flexible and extensible, hence the name
  • sf: the standard package for geospatial work in R – introduces spatial objects and operations
  • tidyverse: a collection of packages used for working with data, all centered around the tidy framework
  • mapgl: an R implementation of the two common OpenGL-based map frameworks, Mapbox GL JS and MapLibre GL JS

Let’s set up some elements using the built-in datasets that come with R and the above packages.

To render the report as docs/index.html, run quarto render index.qmd from the command line in the repo root.

One of the packages in tidyverse is the popular ggplot2, used to create data analysis graphics using the “Grammar of Graphics”. A useful reference is the online book ggplot2: Elegant Graphics for Data Analysis (3e), which was made with Quarto!

The number of diamonds by color and cut in the built in diamonds data set can be visualized using a stacked bar chart. We’ll define the category color manually:

ggplot(data = diamonds,
       aes(color, fill = cut)) +
  geom_bar() + 
  xlab("Color Category") +
  ylab("Count") +
  labs(title = "Diamonds by Color and Cut",
       subtitle = "Source: diamonds (built-in dataset)") +
  scale_fill_discrete(name = "Color Category",
                      palette = c("#f1eef6",
                                  "#bdc9e1",
                                  "#74a9cf",
                                  "#2b8cbe",
                                  "#045a8d"))

Virtually every element of a ggplot graphic can be modified. Let’s say we want to remove the panel shading, leaving only y-axis major grid lines in a light grey, and update the text color to a softer black:

ggplot(data = diamonds,
       aes(color, fill = cut)) +
  geom_bar() + 
  xlab("Color Category") +
  ylab("Count") +
  labs(title = "Diamonds by Color and Cut",
       subtitle = "Source: diamonds (built-in dataset)") +
  scale_fill_discrete(name = "Color Category",
                      palette = c("#f1eef6",
                                  "#bdc9e1",
                                  "#74a9cf",
                                  "#2b8cbe",
                                  "#045a8d")) +
  scale_y_continuous( breaks = seq(0, 30000, by = 5000)) +
  theme(
    panel.background = element_blank(),
    panel.grid.major.y = element_line(color = "grey80"),
    panel.grid.major.x = element_blank(),
    panel.grid.minor.x = element_blank(),
    axis.ticks = element_blank(),
    text = element_text(color = "#141414"),
    axis.text = element_text(color = "#141414")
  )

And so and and so on. LLMs are good with helping with the arcane little tweaks for plots.
